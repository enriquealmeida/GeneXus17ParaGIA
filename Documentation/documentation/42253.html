Counting Type property

Sets if the SD Relative Timer control will make a countdown, count the elapsed time, or both.

Values

Auto
Down
Up

Description

It allows indicating if the control will make a countdown, count the elapsed time, or do both depending on whether the event has already happened or not.

For example, you may want the control to only make a countdown (Counting Type = Down), and once the count reaches zero, the count ends. In this case, suppose the event will happen within 15 minutes; the control will begin to count and will be decreasing the time until reaching 0, then it will stop.

Also, you may want the counter to start only after the event begins (Counting Type = Up). In this case, time will start to be added after the start time of the event; when the event starts, the control will start counting from zero up to the time the event lasts.   

When the variable value is set to Auto, if the event hasn't started yet it behaves as if the property value is set to Down. Once the event begins, the control behaves as if the property value is set to Up.  

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply changes made by this property, Build a main object.

Scope

Platforms: Smart Devices(IOS)
Controls: Attribute/Variable (Control Type: SD Relative Timer)
