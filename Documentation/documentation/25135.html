Web Form Defaults property

Determines the default Web form to be generated (Web Abstract Editor or HTML editor).

Values

Responsive Web Design The object's form will be generated using the Web Abstract Editor. As a result, the design will be responsive.
Previous versions compatible The object's form will be generated using the HTML Editor. This is the default value.

Scope

Objects: Transaction, Web Work With
Generators: Java, .NET, .NET Core

Description

When there are Transaction forms and objects automatically generated by the Work With Pattern, the user can decide whether the web form is going to be initialized using the HTML Editor or the Web Abstract Editor. If the initialization is done using the latter, the resulting design will be responsive.

The property at the Transaction level and at Work With Pattern instance level inherit their values from the value set at the version level. This value can be changed for each Transaction and Pattern instance.

Notes:

  • This property also applies when the menu option Apply Default (for Win and Web Forms) is executed for a web form.
  • When the Web Form Defaults property is set to Responsive Web Design, the following happens in the KB:
    • The Flat (Theme), RwdMasterpage, and RwdRecentLinks objects are consolidated in the KB.
    • The Default Master Page property is set to RwdMasterpage and the Default Theme property is set to Flat.
  • When using a Master Page, keep the Web Panel and associated MasterPage using the same WebFormDefaults property value; do not mix them as unexpected behavior may occur.
