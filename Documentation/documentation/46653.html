GeneXus 17 BPM Suite Release Notes

GeneXus 17 BPM Suite is the most complete and robust version of GXflow that includes performance and scalability improvements and adds support for .NET Core, among others.

There follows a list of the most important improvements and a link to all the details too.

Modeling: New Timers

A process can now start with a Timer Event. Both the Start Event and the Intermediate Timer Events now follow the BPMN 2.0 specification.

More information at  Timer Start Event in BPD, Timer Intermediate Event in BPD.

GXflow on .NET Core

Now the GXflow Client can run on .NET Core, the successor of .NET.

The Supported platforms that have been added are as follows:

  • .NET Core / MySQL
  • .NET Core / Oracle
  • .NET Core / SQL Server
  • .NET Core / PostgreSQL

As you may know, the BPM Suite is completely developed with GeneXus. So it could take advantage of the GeneXus .NET Core Generator to achieve this.

Check the new GXflow Software Requirements.

Technology update

For security reasons and the lifecycle of JRE 1.6 and 1.7, now GXflow for Java runs on Oracle JRE or Open JRE 1.8 or higher.

Also for security reasons, Browser support has been updated, since GXflow's front end now runs on jQuery 3.5.1.

Check the new GXflow Software Requirements.

Robustness, Performance, and scalability improvements

Several performance and scalability aspects have been improved in this version, and new settings for this are available. Also, integration with GeneXus Access Manager (GAM) has been improved; internal license management has been improved as well.

More information at GXflow Maintenance Script

Localization for new regions

As GXflow is being distributed and used in China, the GXflow client has been translated into Chinese.

The standalone Business Process Modeler is now also available in Portuguese.

Licenses Upgrade

This version requires an upgrade of GXflow Licenses to version 17.

Follow these steps to upgrade them:

  1. Use GXflow license manager
  2. If you own nominated licenses, export nominated users, so you don't need to type them in again later
  3. Uninstall licenses
  4. Request a new license
  5. If you own nominated licenses, HowTo: Work with GXflow license manager

More Information: GXflow license scheme, HowTo: Work with GXflow license manager  

GXflow's Database Schema changes

Changes have been made to the database schema in GeneXus 16 upgrade 7, so if you are moving from GeneXus 16 Upgrade 6 or prior to GeneXus 17, some special steps may need to be followed.

More Information: HowTo: Deploying a Workflow-based Application
