Reorganizations associated to Data Views

When working in your Knowledge Base with an external table (associated with a Data View, probably created through the Database Reverse Engineering), GeneXus does not reorganize it (this is informed in the Navigation Report).

In particular when at a certain moment, you delete a Data View, then the next Reorganization will create the external table as "internal".

This implies that:

  • the table will be created in the default Data Store (in the database associated with your application).
  • the data of the external table will be copied from the original Data Store to the new one, preserving the data.


On the other hand, if you do not delete the Data View, but you change its Data Store or platform associated with the Data View, the same will occur:

  • the table will be created in the new specified Data Store.
  • the data of the external table will be copied from the original Data Store to the new one, preserving the data.
