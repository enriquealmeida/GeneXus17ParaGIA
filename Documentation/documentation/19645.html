When to use Query and when to use Data Provider with the QueryViewer control

What is best when using the Query Viewer: a Query object or a Data Provider?

The best choice is to use the Query, except in cases where the complexity or the processing implied for obtaining data do not allow it.

The Query object can solve almost all of the most frequent queries. It is more declarative and its maintenance is easier than that of a Data Provider. In it, we can identify the attributes we want to obtain and the additions or operations to be applied to them.

In some cases the Query object may not be enough, and the database design or the need for a complex logic for obtaining the information may require that we use a Data Provider instead.
