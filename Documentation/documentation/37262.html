Update URL property

Shows the URL of the Store page where the Application is published (e.g. https://play.google.com/store/apps/details?id=<package_name>) or a link to where the APK itself is deployed.

Scope

Objects: Menu, Panel, Work With (Only Main Objects)
Generators: Android

Description

If the application is published in Google Play, you must provide an URL like this:

https://play.google.com/store/apps/details?id=<package_name>
   Where <package_name> <package_name>is the one specified in the Android Package Name property 
   (e.g. https://play.google.com/store/apps/details?id=com.artech.sales1.sales).</package_name>

If the application is published, for example, in Samsung Apps Store or any other stores, this URL must change.

To support multiple stores, you could use the market URI scheme (as follows), but you should consider that the application must be published on them.

market://details?id=<your_app_package_name>

In such a case, the device should display a selector at the bottom of the screen with those stores installed on the device and where the end user can download a new version of the app.

 

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply the corresponding changes when the property value is configured, Build the Main Object.

Availability

This property is available since GeneXus 15 Upgrade 4.
