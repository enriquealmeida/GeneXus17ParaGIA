Knowledge Manager Import

The import process in GeneXus is intended for knowledge-sharing among developers. You import what someone else (or yourself) exported. Changes made to your application by other team members, new developments that need to be included in your application, and existing "application blocks" (see GXopen) are only a few examples of Knowledge Manager Import uses.

The Knowledge Manager Import dialog allows you to:

  1. Select and load an export file to import.
  2. See the objects in the export file.
  3. Select the objects you want to import.
  4. Automatically convert export files from older GeneXus versions and save the converted export file. This conversion is made when a previous version export file is loaded and a message appears telling you that this conversion has been made.
  5. Filter results by import result (i.e., objects with errors, warnings or OK messages).
  6. Specify import options.

The Import process updates the Output window with relevant process status, warnings and errors.

Note that, while importing, Full Text Search indexing is paused (if not already). Once the import process ends, indexing resumes.

Note

When the Automatic Backup option is set to Yes, a Frozen Version is created in the Knowledge Base with the backup remaining visible in the KnowledgeBase Versions tool window.
