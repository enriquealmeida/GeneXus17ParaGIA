Operation and Monitoring the DevOps process

When the operation and monitoring of an application are included in the Continuous Deployment process, with the purpose of generating information that feeds back into the plan of what features should be developed afterward, a DevOps process is achieved.

In order to perform the monitoring tasks of the application in GeneXus, there is an API called Log external object that allows you to monitor the application.

Regarding monitoring, observability must also be taken into account, which is the ability to control what happens within a system so that this control is optimal.
That serves to realize what is happening.

For example, when there is an error that nobody expected, it is because there is no optimal control to know what is happening in the system.
The observability comes here to give indications of how you have to act to have a solution to see what happens within the systems.
