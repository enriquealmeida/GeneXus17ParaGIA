Minimum Time Between Table Purges property

For Offline Native Mobile applications using GeneXus generated synchronization programs, this property determinates the time between calls to the internal maintenance programs to purge the synchronization tables ( 'GXPARAMETERS', 'GXDEVICERESULT' and 'GXRESULTROW').

Values

Value (in seconds) Minimum time lapse the app waits between calls for the Maintenance programs - by default 3600 seconds

Description

The user can set the minimum time interval that needs to pass before the maintenance programs should be called in order to avoid that the Synchronization Tables increment its size. For example, if the value is 1 hour (3.600 seconds), then the Synchronization Tables will be "cleaned" in each hour. And if the Minimum Time Between Receives property is greater than 1 hour, then this process will be triggered each time the synchronization process is done.

This property is available when the Data Receive Granularity property has "By Row" value.

How To Apply Changes

Run of the Main Object for Smart Devices is enough to apply this changes.

Availability

As from GeneXus Salto Beta 3 

Scope

Objects Offline Database object
Platforms Android, Apple iOS
Generators .Net, Java
