Servlet directory property (GeneXus 17 Upgrade 4)

Allows specifying the directory where the servlets generated will be transferred. At compile time of each Web Object, the generator will copy it to the specified directory.

Description

Notes:

  • The path specified must be relative to the client, that is the path seen from the PC in which it is working with GeneXus.
  • The generator will copy, in addition to the .class files, the .cfg files which contain the information of the entered properties; therefore, if some value of the model properties is modified, it is necessary to generate again at least one object and to compile some Web Object. In addition, so that this change is taken into account at runtime, it is necessary to restart the Servlets server.

Values Examples:

  • H:\servlets - When the servlets server is installed on a machine that is mapped with drive H and "servlets" is the name of the folder where the servlets must reside.
  • C:\resin\doc\web_inf\classes â€“ In case of using RESIN locally.

How to apply changes

To apply changes made by this property, do a Build All.
