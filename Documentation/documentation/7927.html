General Description of Procedures

The definition ofÂ procedures will depend on what you want to obtain from the database. The database is created through transaction definitions. Thus, you can use all the knowledge defined. For example, you can use formulas, tables, table relations, extended tables, etc. without the need to define anything else.

It is not necessary to indicate the tables to be used by procedures because the attributes only are enough for GeneXus to determine the tables and indices to be used. This is possible because GeneXus has full knowledge of the database structure and knows what is required by the requesting report.

This is the way to achieve an actually independent database, any change made to a table will be automatically managed by GeneXus.
