Exit command

The Exit command allows you to leave a For Each command (X Evolution 2), Xfor Each command, Do While command or For To Step Command. The next command to be executed after an exit is the command following the EndFor, Xendfor, EndDo or EndFor command corresponding to the For each, Xfor each, Do While or for group containing the Exit command.​

Syntax

Exit

Notes

  • The Exit command is only valid inside a "For Each" or "Xfor Each". It is not valid within a "For Each Line" group.
  • This command does not work within a Do While when generating Cobol for iSeries code.

Example

For instance, in a system that handles Employee information, we may want to know if an employee has quit within a specific period of time. After verifying if there is at least one matching the condition, we want to exit the for each and stop the search.The implementation would be:

&Found = 0
For each order ResignDate
    Where ResignDate >= &StartDate
    Where ResignDate <= &EndDate
         &Found = 1
          Exit
EndFor

If &Found = 1
    Msg('Employee found that satisfies the condition')
Else
    Msg('NO Employee satisfies the condition')
EndIf

Scope

Objects: Procedure object, Web Panel object, Work Panel object
