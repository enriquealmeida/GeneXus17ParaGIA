Deploy Application Targets

The following document lists the available targets to deploy when using the Application Deployment tool.

The default target for every platform is "Local," which would just create the packaged app and tell the user where the package has been created.

Common targets

There are some common targets for every platform, as follows:

Docker: Creates an image with the selected web app based on the selected properties. For more information, read How to Deploy an Application to Docker

AWSEB: Amazon Web Services Elastic Beanstalk. For more information, read How to Deploy an Application to AWS Elastic Beanstalk

AZURE: Microsoft Azure. Deploys the web app as an Azure WebSite. For more information, read Deploy as Azure Web App

Java

AWSLAMBDA: Only deploys Procedures set as main as Lambda functions. For more information, read How to deploy an application to AWS Lambda and AWS API Gateway

IBM Cloud: Deploys to IBM Cloud. For more information, read How to deploy an application to IBM Cloud

GAE: Deploys to Google App Engine. For more information, read How to deploy an application to Google App Engine

HCP: Deploys to SAP Cloud Platform. For more information, read How to deploy an application to SAP Cloud Platform
