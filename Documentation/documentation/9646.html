SetEmpty method

Assigns the empty value in an attribute, variable, or business component depending on its types.

Syntax

attribute|variable.SetEmpty()

Scope

Objects: Procedure, Transaction, Web Panel, Work Panel, Objects for Native Mobile applications development
Generators: .NET, Java, Ruby (up to GeneXus X Evolution 3), RPG, Visual FoxPro (up to GeneXus X Evolution 3), Cobol

Values

Data Type Empty value
Boolean False
Character(N), VarChar(N), LongVarChar(N)   N-blanks
Date 0001-01-01
DateTime 0001-01-01 00:00:00
Numeric 0.0
Time 00:00:00

Samples

OwnerCel.SetEmpty() if OwnerName.IsEmpty();
If Y.IsEmpty()
   X.SetEmpty()
Else
   X = Y
EndIf

Notes:

  • Since GeneXus 15 Upgrade 9, Apple offline apps will consider Date/DateTime empty value as 0001-01-01 insted of 1970-01-01. In case that you use an empty value for displaying data, those records with the old empty date value (1970-01-01) will start to be displayed. The following offline Procedure script will fix the problem in the Offline Database: 
    &oldEmptyDate = ymdtod(1970,1,1)
for each
where InfoDate = &oldEmptyDate
   InfoDate.SetEmpty()
endfor
commit
  • Since GeneXus 16 upgrade 2, this method can be used in Client-side Events in Native Mobile Applications for Apple.
