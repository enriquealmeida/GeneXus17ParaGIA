MySQL version property (GeneXus 16 Upgrade 9 or prior)

Sets the MySQL version range for which the code is generated.

Values

4.x to 5.0.2 The MySQL version installed in the Server is a version between 4.x and 5.0.2.
5.0.3 to 5.7.6 The MySQL version installed in the Server is a version between 5.0.3 and 5.7.6.
5.7.7 to 5.7.23 The MySQL version installed in the Server is from 5.7.7 to 5.7.29.
8.x or higher The MySQL version installed in the Server is 8.x or higher.

Scope

Available at the Data Store.
Data Store: MYSQL

Description

This property is used, for example, to determine the syntax for OUTER JOINs and for table creation/reorganization (i.e. syntax of CREATE TABLE, CREATE INDEX, etc.).

Notes related to MySQL 8 or higher:

  • Although the property can be set to MySQL version 8.x, there are still some known issues related to Geography data types
  • Java-specific: If date fields are read with one day less (eg.: 1930-01-30 is returned as 1930-01-29), you may need to add the timezone of the application server to the connection string (eg.:&serverTimezone=America/Montevideo)
  • As stated in the Release Notes of MySQL 8 the default character_set_server, character_set_database, collation_server, and collation_database environment variables changed. Furthermore, "The pad attribute for Unicode 9.0.0 collations was changed from PAD SPACE to NO PAD." Consequently, spaces are treated at the end of strings like any other character, which means that comparing "A " with "A" does not return true by default. If you do not want this behavior, you should change those defaults.

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply the corresponding changes when the property value is configured, execute a Rebuild All.
