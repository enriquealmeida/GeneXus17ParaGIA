Deploy to cloud property

Facilitates cloud prototyping by deploying the application to the GeneXus cloud.

Values

No This is the default value.
Yes Sets default values to deploy the application to the GeneXus Cloud.

Scope

Generators: Angular, Java, .NET, .NET Core
Level: Generator

Description

By setting this property to "Yes," the Database name, Web root, Deploy Virtual Directory, Deploy Server URL, and Services URL properties are automatically set with default values to deploy the application to the GeneXus cloud. Once the application is run, it is deployed to the GeneXus cloud.

Note

The GAM repository and application are also deployed to the cloud. This means that the GAM database will be created automatically on the cloud.

Steps

  1. Set related properties

    1. Java Specific: The property "Use Annotation for servlet definition" must be set to "No" when deploying to apps2.genexusx.com. The reason is that apps2.genexus.com is hosted by Tomcat 6.0. 

    2. Review affected properties (Database name property, Web Root property, Deploy Virtual Directory property and Deploy Server URL property, Services URL property). You may want to point to other Servers available for Cloud prototyping.

    3. Apple Specific: You may need to configure App Transport Security property group if Services URL property uses SSL. 

    4. Angular Specific: You have to set the Run Target property to GeneXus DeployToCloud.

  2. Create Database

  3. Rebuild all Objects.

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply the corresponding changes when the property value is configured, execute a Rebuild All.
