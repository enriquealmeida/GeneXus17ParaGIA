IsEmpty method

Returns True if the value of a given attribute or variable is empty 1.

Syntax

Attribute | &Variable.IsEmpty()

Type Returned:
Boolean (True or False)

Scope

Objects: Procedure, Transaction, Web Panel
Generators: .NET, Java, Ruby (up to GeneXus X Evolution 3), RPG, Visual FoxPro (up to GeneXus X Evolution 3), Cobol

Description

Using the method with attributes will depend on whether the condition is evaluated on the Server (DBMS) or on the Client (Application Server). On the Client's side, if an attribute is null (or empty), the IsEmpty method returns true (in most situations, GeneXus handles nulls for attributes as expected). On the Server's side, only if the attribute is empty (not null), the IsEmpty method returns true.

Trailing blanks are ignored.

Samples

Case: Server's side evaluation

In this case, the condition is evaluated on the Server's side, so only empty values are retrieved.2

For each
    Where attribute.IsEmpty() // server side execution
          ...  // Only Empty attributes will be considered
EndFor

Case: Client's Side evaluation

This code is evaluated on the Client's side. Both null and empty values are retrieved.

For Each
    If attribute.IsEmpty() // client side execution
       ...   // Empty and Null attributes will be considered
    EndIf
EndFor

2 Some "Where" conditions could be resolved on the Client's side.  This could happen when the navigation group (For Each, Data Provider, Grid) includes in their Where/Condition clause a function or method that can not be evaluated by the DBMS.
