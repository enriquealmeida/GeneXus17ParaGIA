Domains with Special Semantics

Some standard domains have special semantics that produce special behavior on fields based on these domains when executed in Smart Devices.

For example, let's suppose that a field is based on the Address domain and that the field is shown in readonly mode on a Smart Device. The application, when the user clicks on a balloon shown beside the field, will redirect to a Google map showing that location.

Domains with special semantics are:

  • Address domain
  • Component domain
  • Email domain
  • Feed domain
  • Geolocation domain
  • HTML domain
  • Phone domain
  • URL domain
  • Time domain
