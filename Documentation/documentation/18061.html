Link Intermediate Event in BPD

This event is always used in pairs throw-catch, being necessary that both events have the same name in order to link them. Its aim is to create a virtual sequence flow.

Symbol

Description

It only can exist one catch event, but many throw events with the same name. All of them are used in the same process.
In order to communicate these events is highly recommended the use of signals.

Scope

Objects: Business Process Diagram
