Maximum workFile lines property

Configures the maximum grid size. The grids in a PC Environment are loaded in a temporary file.

Description

Values:

Any positive integer
 
Default Value = 10000

Notes:

  • For 3-tier java applications, this property is not supported when the work panel does not have a base table and the property “Execute load event in application server” = YES.

  • The maximum Grid size in the iSeries is 9,999 lines, however.

  • There are no restrictions when PC or LANs are used. This freedom may cause problems, for example, if you are working with a Work Panel or Transaction that loads a 300,000-record base table and the user goes to the end of the file (Ctrl-End). In this case, a temporary file containing 300,000 records will be generated.

  • When the limit is reached while working with a Work Panel the following message will be displayed:

          Number of lines exceeded xxxx

    And it allows you to continue working, but no other lines will be loaded.

How to apply changes

To apply changes made by this property, do a Build with this Only of the object.

Scope

Platforms: Web(.Net, Java)
