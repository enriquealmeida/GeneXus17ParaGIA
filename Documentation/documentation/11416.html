Workflow Context

As of GeneXus X it is not necessary to add the parm rule with the three parameters in all the objects, it is only required when working with procedures associated to conditions. If you want to work with the proccess definition, process instance or the workitem, use the WorkflowContext variable (don´t need to define it): 

  • &WorkflowContext.ProcessDefinition (data type: WorkflowProcessDefinition)
     
  • &WorkflowContext.ProcessInstance (data type: WorkflowProcessInstance)
     
  • &WorkflowContext.Workitem (data type: WorkflowWorkitem)

If you want to use a procedure associated to a condition instead of the condition editor you have to define the following parm rule:

parm(in:&wfprocessdefinition,in:&wfprocessinstance,in:&wfworkitem,out:&conditioncode);


 

Where the variables are defined as follows:

&wfprocessdefinition - WorkflowProcessDefinition
&wfprocessinstance - WorkflowProcessInstance
&wfworkitem - WorkflowWorkitem
&conditioncode - Numeric(4.0)

Warning: you can't use the &WorkflowContext in procedures associated with conditionals, timers events and event handling properties.
