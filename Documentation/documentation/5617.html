Automatic Backup property

Controls whether a backup is made before importing.

Values

False
True

Description

Default value: True.

Backups are always a good way of recovering from errors. Experience tells us that importing is a common source of errors (inconsistent Export files, missing objects, etc.).

The backup process consist in creating a Frozen Version with the status of the current Development Version, which will be visible and available to make a Revert operation in the Knowledge Base Versions Tool Window.

Scope

Action: Import option
