Model Automation

This stage in the BPM Cycle, is where the model becomes a functional application.

To make the diagram executable, we should associate GeneXus objects to the diagram symbols. For example, an interactive task may be associated to a Transaction object, while a script task may be integrated with a Procedure object.

GeneXus procedures also could be associated with Gateways to evaluate a complex decision, to a Connector, for specific conditional flow behaviour, or to a Conditional Event, which trigger with a conditional expression that must be met for the process.

To associate a GeneXus object to a task, we have to use the task Application Property allows indicating the GeneXus object that will be invoked upon executing the task.

To know the details on using this property, and see a practical example on model automation, please have a look to My first BPM Application
