Xnew Command

The Xnew command inserts a record in an external file (base table). The insertion is performed when the Xendnew is found. 

Syntax

Xnew ‘filename’
            {att = exp}…
Xendnew

Where:
filename
The name of the Data View object defined to access the external file.

att
The attribute to be assigned.

exp
The expression assigned can be either an Attribute, a Variable, a Constant, a Function or an Arithmetic Expression (Date and Numeric types only).

Notes

  • Client/Server: Duplicate keys are checked only for Data Views where the index has been defined as unique.  
  • iSeries: External logical file keys must be defined as unique and uniqueness is automatically verified by the operating system.

Example

Given the following Data View with no associated table:

Name: CliExt
Composition: 
                        CliId
                        CliName
Indices: Name: ICliExt (Unique)
Composition: CliId

The following example shows a procedure definition that inserts a new Client:.

Xnew 'CliExt'
    CliId = 1
    CliName = 'Ulrich Karnbatz'
Xendnew

If &Err = 1
    Print Error
Endif

If the client number already exists the message 'Record already exists' will be displayed. Duplicate keys will be checked because the index was defined as unique.

Scope

Objects: Procedure object
