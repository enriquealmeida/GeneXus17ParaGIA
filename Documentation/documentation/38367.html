Methodology

GXtest v4 is a completely redesigned product to encourage GeneXus community to adopt and boost the development process by adopting DevOps practices and Agile methodologies designed for GeneXus developers, QE and DevOps engineers.

Based on the pipeline concept for building software, GXtest adds different automation layers for each step and defines a methodology to achieve this in GeneXus.

Follow next sections to understand how this tool fits into the pipeline concept:

  • Typical GeneXus development cycle for Agile
  • Testing Automation Pyramid (ROI-based automation)
  • Continuous integration with Jenkins
